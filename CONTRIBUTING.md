# 政策提案の参加ガイド

このガイドでは、政策提案の方法と議論への参加方法を説明します。

## GitHubが初めての方へ

GitHubは、提案の管理・議論・改善の履歴を透明に残せるツールです。
プログラミングの知識は不要です。

### アカウント作成

1. [github.com](https://github.com) にアクセス
2. 「Sign up」をクリック
3. メールアドレス、パスワード、ユーザー名を入力
4. メール認証を完了

### 操作がわからない場合

Discordサーバーの `#github-help` チャンネルで質問してください。サポートします。

## 提案の全体フロー

提案の進行状況は **[Projects ボード](https://github.com/users/kkhassou/projects/2)** で確認できます。

```
Issue作成 → コミュニティ議論 → 提案書作成（PR） → レビュー → マージ → 議員提出
[アイデア]    [検討中]          [精査中]                      [議員提出準備] [提出済]
```

| ステージ | 場所 | 誰が参加？ | ボード上の列 |
|---------|------|-----------|-------------|
| 提案 | Issue | 誰でも | アイデア |
| 議論・投票 | Issue コメント・👍 | 誰でも | 検討中 |
| 提案書作成 | ブランチ → PR | 提案者 or 有志 | 精査中 |
| レビュー・改善 | PR コメント | 誰でも | 精査中 |
| 完成・マージ | PR マージ | レビュアー | 議員提出準備 |
| 議員提出 | 運営 | 運営 | 提出済 |
| 実現 | - | - | 実現済 |

## 政策を提案する

### Step 1: Issue を作成する

1. [Issues タブ](../../issues) を開く
2. 「New Issue」をクリック
3. 「政策提案」テンプレートを選択
4. 各項目を記入して「Submit new issue」をクリック

これであなたの提案が公開されます。自動的に Projects ボードにも追加されます。

### 良い提案のポイント

- **具体的に書く**: 「教育を良くする」ではなく「小学校の1クラスあたりの人数を30人以下にする」
- **課題を明確に**: なぜその政策が必要なのかを説明する
- **データがあれば添える**: 根拠となる統計や事例があると説得力が増します
- **実現可能性を意識する**: 完璧でなくてOK、議論の中で磨かれます

## 議論に参加する

### コメントで意見を伝える

- 既存の提案（Issue）を開いてコメントを書く
- 賛成・反対だけでなく「なぜそう思うか」を書くと議論が深まります

### リアクションで賛同を示す

- 提案に :+1:（👍）を押すと賛同を表明できます
- 賛同数が多い提案は優先的にブラッシュアップされます

### Discussionsでアイデアを共有

- まだ具体的な提案になっていないアイデアは [Discussions](../../discussions) へ
- 「こんな課題がある」「こういう方向性はどうか」といった議論ができます

## 提案書の作成に貢献する（ブランチ・PR）

賛同が集まった提案は、詳細な提案書にまとめます。
ブランチを作って提案書を書き、Pull Request（PR）で提出します。

### ブランチの作り方

#### GitHub Web UIで作成する場合（初心者向け）

1. 提案書を作りたい Issue を開く
2. 右サイドバーの「Development」→「Create a branch」をクリック
3. ブランチ名を入力（下記の命名規則を参照）
4. 「Create branch」をクリック
5. Web UI上で「Add file」→「Create new file」で提案書を作成
6. ファイルパス: `proposals/分野名/issue番号-タイトル.md`
7. 内容を記入して「Commit changes」

#### コマンドラインの場合

```bash
git checkout -b proposal/123-childcare-support
# 提案書を作成
git add proposals/education/123-childcare-support.md
git commit -m "提案書作成: 子育て支援の拡充 #123"
git push -u origin proposal/123-childcare-support
```

### ブランチ命名規則

| パターン | 用途 | 例 |
|---------|------|-----|
| `proposal/Issue番号-短い説明` | 提案書の作成・精査 | `proposal/12-childcare-support` |
| `docs/説明` | ガイド等ドキュメントの改善 | `docs/update-contributing` |

### Pull Request の出し方

1. ブランチにファイルをコミットした後、リポジトリのページを開く
2. 「Compare & pull request」ボタンが表示されるのでクリック
3. タイトル: `[提案書] ○○に関する提案 #Issue番号`
4. 本文: 提案の概要、変更点を簡潔に記載
5. 「Create pull request」をクリック

### 提案書のファイル配置

```
proposals/
├── economy/          # 経済・財政
│   └── 5-tax-reform.md
├── education/        # 教育・子育て
│   └── 12-childcare-support.md
├── healthcare/       # 医療・福祉
├── digital/          # デジタル・行政改革
├── party-operations/ # 党内運営
├── other/            # その他
└── template.md       # テンプレート
```

ファイル名: `Issue番号-短い説明.md`（例: `12-childcare-support.md`）

### PR上でのレビュー

- PRが作成されると、誰でもコメントで意見を書けます
- 「Files changed」タブで提案書の具体的な箇所にコメントできます
- 修正が必要な場合は、ブランチに追加コミットするだけでPRに反映されます

### 提案書のテンプレート

`proposals/template.md` をコピーして使ってください。主な項目：

- 概要
- 背景・課題
- 提案内容（具体的な施策、実施方法）
- 期待される効果
- 想定される課題・リスク
- 参考事例
