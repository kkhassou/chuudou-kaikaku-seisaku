# 政策提案の参加ガイド

このガイドでは、政策提案の方法と議論への参加方法を説明します。

## GitHubが初めての方へ

GitHubは、提案の管理・議論・改善の履歴を透明に残せるツールです。
プログラミングの知識は不要です。

### アカウント作成

1. [github.com](https://github.com) にアクセス
2. 「Sign up」をクリック
3. メールアドレス、パスワード、ユーザー名を入力
4. メール認証を完了

### 操作がわからない場合

Discordサーバーの `#github-help` チャンネルで質問してください。サポートします。

## 政策を提案する

### Issue（提案）の作成手順

1. [Issues タブ](../../issues) を開く
2. 「New Issue」をクリック
3. 「政策提案」テンプレートを選択
4. 各項目を記入して「Submit new issue」をクリック

### 良い提案のポイント

- **具体的に書く**: 「教育を良くする」ではなく「小学校の1クラスあたりの人数を30人以下にする」
- **課題を明確に**: なぜその政策が必要なのかを説明する
- **データがあれば添える**: 根拠となる統計や事例があると説得力が増します
- **実現可能性を意識する**: 完璧でなくてOK、議論の中で磨かれます

## 議論に参加する

### コメントで意見を伝える

- 既存の提案（Issue）を開いてコメントを書く
- 賛成・反対だけでなく「なぜそう思うか」を書くと議論が深まります

### リアクションで賛同を示す

- 提案に :+1:（👍）を押すと賛同を表明できます
- 賛同数が多い提案は優先的にブラッシュアップされます

### Discussionsでアイデアを共有

- まだ具体的な提案になっていないアイデアは [Discussions](../../discussions) へ
- 「こんな課題がある」「こういう方向性はどうか」といった議論ができます

## 提案の改善に貢献する

### 既存の提案を改善する

1. 改善したい提案のIssueを開く
2. コメントで具体的な改善案を提示する
3. データや事例を追加で提供する

### 提案書の作成を手伝う

精査フェーズに進んだ提案は、`proposals/` フォルダに詳細な提案書としてまとめます。
Pull Request を使って提案書の作成・修正に参加できます。

## ラベルの意味

| ラベル | 意味 |
|--------|------|
| `経済・財政` | 経済分野の提案 |
| `教育・子育て` | 教育分野の提案 |
| `医療・福祉` | 医療分野の提案 |
| `デジタル・行政改革` | DX分野の提案 |
| `その他` | その他の分野 |
| `検討中` | コミュニティで議論中 |
| `精査中` | 有志チームがブラッシュアップ中 |
| `議員提出済` | 議員に提出された提案 |
| `実現済` | 実現した提案 |

## 困ったときは

- GitHubの操作: Discord `#github-help` チャンネル
- 提案の書き方: Discord `#proposal-drafting` チャンネル
- その他の質問: Discord `#free-talk` チャンネル
